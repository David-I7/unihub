# Reframe the advanced Contribute page

## Parent

.scratch/student-suggestions-contextual-front-door/PRD.md

## What to build

Update the advanced `Contribute` page so it remains visible in main navigation but is clearly framed as a maintainer and advanced contribution surface. Students who want to fix a specific course item should be directed back to Course page `Suggest update` actions.

The page should keep GitHub issue and pull request assist modes, with GitHub issue as the default.

## Acceptance criteria

- [ ] The `Contribute` page starts with the heading `Maintainer contributions`.
- [ ] The page explains that it is for repository-level course data changes, advanced corrections, and preparing GitHub issues or pull requests for maintainer review.
- [ ] The page tells students fixing a specific course item to start from that Course page and use `Suggest update`.
- [ ] The advanced page remains accessible from main navigation.
- [ ] The advanced page keeps GitHub issue and pull request assist modes.
- [ ] GitHub issue mode remains the default.
- [ ] Existing advanced contribution behavior continues to validate and generate issue or pull request content.
- [ ] Tests or browser verification cover the updated top-of-page framing.
